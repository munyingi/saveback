# SaveBack

A save-for-save network. People exchange WhatsApp numbers to grow their status
views and reach. A number stays hidden until both people save each other; on a
mutual save it unlocks with a copy button and a one-tap WhatsApp deep link that
opens from the user's **own** phone.

- **Stack:** Next.js (App Router, TypeScript) · Supabase (Postgres, Auth, RLS) ·
  Tailwind · installable PWA · Cloudflare hosting.
- **Security model:** phone numbers are consent-gated in the database. A phone is
  readable only by its owner or a matched user, enforced by row-level security —
  not just the UI. See `sample/schema.sql`.

## Getting started

1. **Install dependencies**

   ```bash
   npm install
   ```

2. **Configure environment**

   ```bash
   cp .env.local.example .env.local
   ```

   Fill in your Supabase project URL and anon key. The service-role key and ban
   salt are server-only — never prefix them with `NEXT_PUBLIC`.

3. **Run the dev server**

   ```bash
   npm run dev
   ```

   Open http://localhost:3000.

## PWA / installability

- The web app manifest is generated by `app/manifest.ts` and served at
  `/manifest.webmanifest`.
- The service worker (`public/sw.js`) registers in **production only**, so the
  cache never masks code changes during development.
- Icons are generated with no dependencies:

  ```bash
  npm run icons
  ```

- To verify install + full-screen behaviour, build and serve a production
  bundle (a service worker needs a production build, plus HTTPS or localhost):

  ```bash
  npm run build && npm start
  ```

  Then use the browser's "Install app" / "Add to Home Screen".

## Build order

This is being built in the order set by the brief, pausing for testing after
each step:

1. ✅ Scaffold (Next.js + TS + Tailwind + PWA + Supabase clients)
2. Apply `sample/schema.sql` to Supabase, confirm RLS
3. Auth: phone OTP + email verification (both required to enter)
4. Onboarding + profile
5. Discover feed via `feed()` RPC
6. Save + match reveal
7. Requests + Matches tabs
8. Boost + You tab
9. Invite: `/u/[handle]` + link + QR
10. Safety: report, block, ban gate, signup denylist

## Non-negotiables

1. Sending is deep-link only — no server ever sends WhatsApp messages.
2. Phone numbers are gated in the DB; never exposed before a mutual match.
3. Accounts require a verified phone **and** a verified email. Email is private.
4. No fake, seed, or bot profiles. Real signed-in users only.
5. Free at launch — boost/intro-pack features exist but are ungated.
